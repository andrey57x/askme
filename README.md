# AskMe — веб-сервис вопросов и ответов

Проект разработан в рамках курса «Веб-технологии» (1-й семестр) в Образовательном центре VK при МГТУ им. Н.Э. Баумана. 

Сервис представляет собой аналог платформы вопросов и ответов (по типу StackOverflow) с поддержкой авторизации, системы оценок вопросов и ответов, тегов и real-time уведомлений.

## Стек технологий
* Основной фреймворк: Python (Django)
* СУБД: PostgreSQL
* Кэширование: Memcached / Redis
* Сервер real-time уведомлений: Centrifugo (WebSockets для мгновенной доставки уведомлений на клиенте)
* Веб-сервер / Reverse Proxy: Nginx
* WSGI-сервер: Gunicorn
* Фронтенд: HTML, CSS, JavaScript

## Реализованный функционал
* **Пользователи и авторизация:** регистрация, авторизация, кастомные профили пользователей с возможностью загрузки аватаров.
* **Вопросы и ответы:** публикация вопросов с тегами, написание ответов, пагинация списков, фильтрация по новизне и популярности.
* **Интерактивные элементы (AJAX):** оценка вопросов и ответов (лайки/дизлайки), выбор автором лучшего ответа на свой вопрос без перезагрузки страницы.
* **Мгновенные уведомления:** отправка real-time сообщений о новых ответах автору вопроса через WebSocket-соединение с Centrifugo.
* **Оптимизация производительности:** раздача статических файлов средствами Nginx, оптимизация запросов к базе данных (select_related, prefetch_related), кэширование медленных блоков страниц.

## Структура проекта
* `app/` — основное Django-приложение (модели, представления, формы, менеджеры запросов).
* `askme_safronov/` — конфигурационная папка Django-проекта (настройки, маршрутизация).
* `nginx/` — конфигурационные файлы веб-сервера Nginx.
* `centrifugo/` — конфигурация для WebSocket-сервера Centrifugo.
* `templates/` — HTML-шаблоны страниц веб-сервиса.
* `static/` — статические файлы (стили, скрипты, шрифты).
* `media/` — пользовательский медиа-контент (аватары пользователей).
* `test/` — тесты производительности и функционала.
* `manage.py` — утилита управления Django-проектом.

## Инструкция по запуску

### Предварительные требования
Для работы проекта локально необходимы установленный Python версии 3.10+ и СУБД PostgreSQL. Для работы в production-окружении требуются настроенные Nginx и Centrifugo.

### Пошаговый запуск в режиме разработки
1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/andrey57x/askme.git
   cd askme
   ```

2. Создайте и активируйте виртуальное окружение:
   ```bash
   python -m venv venv
   source venv/bin/activate  # Для Linux/macOS
   # venv\Scripts\activate  # Для Windows
   ```

3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

4. Примените миграции базы данных:
   ```bash
   python manage.py migrate
   ```

5. Запустите встроенный сервер разработки Django:
   ```bash
   python manage.py runserver
   ```
